Webchat Alpha

What's this?

This is the latest deployed version of the GAAP Webchat Alpha user research prototype.

The source code for this is on GitHub.

Instructions

This works best with two devices, or you can also try two browser windows.

Pick one service and appropriate agent interface from the links below, and open each. For example, you can pick the Dart charge prototype, for which you'll need the Custom Chat agent interface.

Custom chat specific

In the custom agent interface, follow the instructions by picking your adviser and writing an optional welcome message. Then click the "Start the chat" button.

In the service prototype, find and click the webchat call to action. Type in your name, and you'll be taken to the queue.

The agent interface will now give you an option to advance the queue, manually. Click 10 times to reach the conversation.

When you're ready to end the conversation, the agent can do it in one click, or the user can also do it by using the top right "end chat" button.

LiveChat specific

You'll need Chris Heathcote's livechat credentials to log into their agent interface. Once there, find the chat section.

In the service prototype, find and click the webchat call to action. Type in your name, and the conversation will begin.

Service links

DVLA address change prototype - Uses LiveChat. Does not validate any of the information requested. Will always fail to find your address, and display the support options, presenting the webchat call to action.

Pay Dartford crossing charge prototype - Uses the custom chat. Does not validate any of the information. Will always fail to submit payment. The webchat call to action is in the persistent footer.

State pension prototype - Uses the custom chat. Validates typing anything at all into the NINO field. Stops at the postcode page. The webchat call to action is in the "I cannot find my NINO" section.

Caveats

This is a prototype built for the sole purpose of user testing in a lab with controlled devices. It is not exhaustively tested and uses plenty of shortcuts. The browsers we used were Chrome for macOS and Windows, Safari on iOS 9, and Chrome for Android.

The prototype does not work with more than one user and one agent. You can have more than one chat in a row, but every chat needs to end before a new one can begin. If you manage to break it and desync the agent/user, refresh both starting with the agent and start over.

If you're chatting as a user and you accidentally refresh the interface, you'll resume where you left off. If you refresh as an agent, you'll become desynced and you'll have to start over.

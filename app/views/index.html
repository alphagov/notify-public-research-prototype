Webchat alpha prototype

What's this?

This is a prototype implementation of a webchat client. It was created for carrying out user research for the GDS alpha project looking at webchat.

The source code for this is on GitHub.

Instructions

This works best with two devices, or you can use two browser windows.

Open the agent interface and follow the instructions by picking your adviser and writing an optional welcome message. Then click the "Start the chat" button.

In the service prototype, find and click the webchat call to action. Type in your name, and you'll be taken to the queue.

The agent interface will now give you an option to advance the queue, manually. Click 10 times to reach the conversation.

When you're ready to end the conversation, the agent can do it in one click, or the user can also do it by using the top right "end chat" button.

We were testing use of webchat with users in Goverment services.
These are not real screens from real services: they helped get the user thinking about a task, instead of webchat or support.

Pay Dartford crossing charge prototype
The task was to pay for using the crossing today.
The prototype does not validate any of the information.
This will always fail to submit the payment.
The webchat call to action is in the footer of all pages.

State pension prototype
The task was to get a state pension forecast.
As a part of this, we made users find their National Insurance number.
Validates typing anything at all into the National Insurance number field. Stops at the postcode page.
The webchat call to action is in the "I cannot find my National Insurance number" section.

Warning!

This is a prototype built for the sole purpose of user testing in a lab. It is not exhaustively tested on all browsers. The browsers we used were Chrome for macOS and Windows, Safari on iOS 9, and Chrome for Android.

The prototype does not work with more than one user and one agent at a time. If someone else is using this prototype, you will see their chats or it may not work properly. You can have more than one chat in a row, but every chat needs to end before a new one can begin. If you manage to break it and desync the agent/user, refresh both starting with the agent and start over.

If you're chatting as a user and you accidentally refresh the interface, you'll resume where you left off. If you refresh as an agent, you'll become desynced and you'll have to start over.
